> **참고**: 특허청 공식 양식 기준 (https://www.patent.go.kr)
> 모범명세서 다운로드: 특허로 → 출원/등록안내 → 모범명세서

## 섹션별 작성 가이드

### 1. 발명의 명칭

**작성 원칙:**
- 발명의 내용을 명확히 표현
- 15~20자 내외 권장
- 상표명, 고유명사 사용 금지
- "~장치", "~방법", "~시스템" 등으로 끝맺음

**예시:**
```
[나쁜 예] 혁신적인 AI 앱
[좋은 예] 행동과학 이론 기반 실행 계획 생성 장치 및 방법
```

---

### 2. 기술분야

**작성 원칙:**
- 1~2문장으로 간결하게
- IPC 분류와 연관되게 작성

**템플릿:**
```
본 발명은 [기술 분야]에 관한 것으로, 더욱 상세하게는 [구체적 기술]에 관한 것이다.
```

**예시:**
```
본 발명은 인공지능 기반 일정 관리 시스템에 관한 것으로,
더욱 상세하게는 사용자의 행동 패턴을 분석하여
최적의 실행 계획을 자동 생성하는 장치 및 방법에 관한 것이다.
```

---

### 3. 발명의 배경이 되는 기술

**작성 원칙:**
- 종래 기술 2~3가지 언급
- 각 기술의 문제점 지적
- 본 발명의 필요성으로 연결

**구조:**
```
[종래 기술 1 설명]
→ [문제점 1]

[종래 기술 2 설명]
→ [문제점 2]

따라서, [본 발명이 해결하고자 하는 문제]를 해결할 수 있는
새로운 기술이 요구되고 있다.
```

**주의사항:**
- 선행기술 검색 결과 반영
- 특정 제품명/회사명 사용 자제
- 논문 인용 시 출처 명시

---

### 3-1. 선행기술문헌 (필수)

> ⚠️ **변리사 실무 기준**: 선행기술문헌은 **1개만** 기재하고, **본 발명과 가장 상이한 것**을 선택하세요.
> 유사한 선행기술을 많이 기재하면 심사관이 진보성흠결(특허법 제29조 제2항) 거절 근거로 사용합니다.

**특허문헌 (1개 권장):**
```
【특허문헌】
(특허문헌 1) 한국등록특허 제10-XXXXXXX호 "발명의 명칭"
```

**비특허문헌 (필요시만):**
```
【비특허문헌】
(비특허문헌 1) 저자명 (연도). "논문/서적 제목"
```

**선행기술 선택 기준:**
- ✅ 본 발명과 **가장 다른** 기술 분야의 특허
- ✅ 차별점이 **명확히 드러나는** 특허
- ❌ 본 발명과 **유사한** 특허 (거절 근거로 사용됨)
- ❌ 여러 개 나열 (많을수록 불리)

**KIPRIS 검색:**
- https://www.kipris.or.kr 에서 키워드 검색
- 유사 특허 확인 후, **가장 다른 것** 1개만 기재

---

### 4. 해결하려는 과제

**작성 원칙:**
- 배경기술의 문제점과 연결
- 본 발명의 목적 명시
- 2~3개 과제로 구분 가능

**템플릿:**
```
본 발명은 상기와 같은 문제점을 해결하기 위하여 안출된 것으로서,
본 발명의 목적은 [과제 1]을/를 제공하는 것이다.
본 발명의 다른 목적은 [과제 2]을/를 제공하는 것이다.
```

---

### 5. 과제 해결 수단

**작성 원칙:**
- 청구항의 내용을 풀어서 설명
- 기술적 구성요소 나열
- "~하는 단계", "~하는 수단" 형식

**구조:**
```
상기 목적을 달성하기 위한 본 발명에 따른 [발명명칭]은/는,
[구성요소 1]과;
[구성요소 2]와;
[구성요소 3]을 포함하는 것을 특징으로 한다.
```

**예시:**
```
상기 목적을 달성하기 위한 본 발명에 따른 실행 계획 생성 시스템은,
사용자의 행동 패턴 데이터를 수집하는 데이터 수집부와;
상기 행동 패턴 데이터를 분석하여 최적 실행 시점을 예측하는 분석부와;
행동과학 이론에 기반하여 실행 계획을 생성하는 계획 생성부와;
상기 최적 실행 시점에 상기 실행 계획을 사용자에게 제공하는 출력부를
포함하는 것을 특징으로 한다.
```

### 6. 발명의 효과

**작성 원칙:**
- 종래 기술 대비 개선점
- **정성적 표현 사용** (향상된다, 개선된다, 가능하다)
- 각 과제에 대응하는 효과

> ⚠️ **변리사 실무 기준**: 효과란에 **"25% 향상"** 같은 **수치 기재 금지**
> - 수치 효과가 실제로 안 나오면 권리행사 어려움
> - 특허권 범위는 효과가 아닌 **과제해결원리(=구성)**로 판단
> - 수치는 【발명을 실시하기 위한 구체적인 내용】의 **실시예**에서 기재

**템플릿:**
```
본 발명에 따르면, 다음과 같은 효과가 있다.

첫째, [정성적 효과 1 - "~가 향상된다"]
둘째, [정성적 효과 2 - "~가 가능하다"]
셋째, [정성적 효과 3 - "~에 기여할 수 있다"]
```

**좋은 예 vs 나쁜 예:**
```
❌ 나쁜 예: "동작 인식 정확도가 25% 향상되었다"
✅ 좋은 예: "동작 인식 정확도가 향상된다"

❌ 나쁜 예: "학습 효율이 35% 개선되었다"
✅ 좋은 예: "학습 효율이 개선된다"
```

---

### 7. 도면의 간단한 설명

**작성 원칙:**
- 각 도면 번호별 설명
- 도면이 무엇을 나타내는지 명시

**템플릿:**
```
도 1은 본 발명의 일 실시예에 따른 시스템의 전체 구성도이다.
도 2는 본 발명의 일 실시예에 따른 처리 흐름도이다.
도 3은 본 발명의 일 실시예에 따른 사용자 인터페이스 예시도이다.
```

---

### 8. 발명을 실시하기 위한 구체적인 내용

**작성 원칙:**
- 가장 상세하게 작성하는 부분
- 도면 참조하며 설명
- 다양한 실시예 포함 가능
- 청구항 범위를 뒷받침
- **수치/정량적 효과는 이 섹션의 실시예에서 기재**

> ⚠️ **변리사 실무 기준 - 표현 규칙**
>
> | 피해야 할 표현 | 권장 표현 |
> |-------------|---------|
> | 바람직한 실시예 | 일 측면에 따른 일 실시예 |
> | ~한다 | ~할 수 있다 |
> | 바람직하게는, 특히 | 기재 안함 |
> | x, y, z 좌표 | 도면 참조 + 정의 추가 |
>
> **이유**: "바람직하게" 사용시 권리행사 때 "명시 안 한 것은 바람직하지 않은 것 아니냐" 문제 발생

**구조:**
```
이하, 첨부된 도면을 참조하여 본 발명의 일 측면에 따른
일 실시예를 상세히 설명하기로 한다.

[시스템 구성 설명]
도 1을 참조하면, 본 발명에 따른 시스템(100)은...

[각 구성요소 상세 설명 - "~할 수 있다" 사용]
데이터 수집부(110)는 ~를 수집할 수 있다.
분석부(120)는 ~를 분석할 수 있다.
계획 생성부(130)는 ~를 생성할 수 있다.

[동작 설명]
도 2를 참조하면, 먼저 단계 S210에서...

[변형 실시예]
다른 실시예에서, 상기 분석부는...

[실험 결과 - 수치는 여기에 기재]
100명을 대상으로 실험한 결과, 정확도가 72%에서 90%로 향상되었다.
```

**실시예 작성 형식:**
```
❌ 나쁜 예 (번호 나열):
1. 입력: 30fps 영상
2. 전처리: 0.5초 단위 분할
3. 결과: 정확도 0.75

✅ 좋은 예 (문장으로 기재):
입력값이 30fps 영상인 경우, 전처리부에서 0.5초 단위로 분할하면,
정확도 0.75의 결과를 도출할 수 있다.
```

## 작성 팁

1. **청구항 먼저**: 청구항을 먼저 작성하고, 명세서에서 뒷받침
2. **도면 활용**: 복잡한 기술은 도면으로 설명
3. **일관된 용어**: 동일한 구성요소는 동일한 명칭 사용
4. **넓은 범위**: 실시예를 다양하게 하여 권리범위 확보
5. **선행기술 인용**: 배경기술에 선행기술 검색 결과 반영
