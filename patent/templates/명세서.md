# 특허 명세서 작성 템플릿

> **참고**: 특허청 공식 양식 기준 (https://www.patent.go.kr)
> 모범명세서 다운로드: 특허로 → 출원/등록안내 → 모범명세서

## 기본 구조 (특허청 공식 양식)

```
【발명의 명칭】
[발명을 간결하고 명확하게 표현하는 명칭]
{English Title}

【기술분야】
[발명이 속하는 기술 분야]

【발명의 배경이 되는 기술】
[종래 기술의 설명 및 문제점]

【선행기술문헌】
  【특허문헌】
  (특허문헌 1) 한국공개특허 제10-20XX-XXXXXXX호
  【비특허문헌】
  (비특허문헌 1) 논문/서적 등

【발명의 내용】
  【해결하려는 과제】
  [본 발명이 해결하고자 하는 기술적 과제]

  【과제의 해결 수단】
  [과제를 해결하기 위한 기술적 수단]

  【발명의 효과】
  [본 발명의 기술적 효과]

【도면의 간단한 설명】
[첨부 도면에 대한 설명]

【발명을 실시하기 위한 구체적인 내용】
[발명의 상세한 실시예 - 최소 1개 이상]

【부호의 설명】
[도면에 사용된 부호 설명]

【청구범위】
[별도 파일로 작성 - 청구항.md 참조]

【요약서】

[별도 첨부]
```

## 섹션별 작성 가이드

### 1. 발명의 명칭

**작성 원칙:**
- 발명의 내용을 명확히 표현
- 15~20자 내외 권장
- 상표명, 고유명사 사용 금지
- "~장치", "~방법", "~시스템" 등으로 끝맺음

**예시:**
```
[나쁜 예] 혁신적인 AI 앱
[좋은 예] 행동과학 이론 기반 실행 계획 생성 장치 및 방법
```

---

### 2. 기술분야

**작성 원칙:**
- 1~2문장으로 간결하게
- IPC 분류와 연관되게 작성

**템플릿:**
```
본 발명은 [기술 분야]에 관한 것으로, 더욱 상세하게는 [구체적 기술]에 관한 것이다.
```

**예시:**
```
본 발명은 인공지능 기반 일정 관리 시스템에 관한 것으로,
더욱 상세하게는 사용자의 행동 패턴을 분석하여
최적의 실행 계획을 자동 생성하는 장치 및 방법에 관한 것이다.
```

---

### 3. 발명의 배경이 되는 기술

**작성 원칙:**
- 종래 기술 2~3가지 언급
- 각 기술의 문제점 지적
- 본 발명의 필요성으로 연결

**구조:**
```
[종래 기술 1 설명]
→ [문제점 1]

[종래 기술 2 설명]
→ [문제점 2]

따라서, [본 발명이 해결하고자 하는 문제]를 해결할 수 있는
새로운 기술이 요구되고 있다.
```

**주의사항:**
- 선행기술 검색 결과 반영
- 특정 제품명/회사명 사용 자제
- 논문 인용 시 출처 명시

---

### 3-1. 선행기술문헌 (필수)

**특허문헌:**
```
【특허문헌】
(특허문헌 1) 한국등록특허 제10-XXXXXXX호 "발명의 명칭"
(특허문헌 2) 한국공개특허 제10-20XX-XXXXXXX호
(특허문헌 3) 미국특허 제US X,XXX,XXX호
```

**비특허문헌:**
```
【비특허문헌】
(비특허문헌 1) Gollwitzer, P. M. (1999). "Implementation intentions"
(비특허문헌 2) BJ Fogg (2019). "Tiny Habits"
```

**KIPRIS 검색으로 확보:**
- https://www.kipris.or.kr 에서 키워드 검색
- 유사 특허의 출원번호/등록번호 기재

---

### 4. 해결하려는 과제

**작성 원칙:**
- 배경기술의 문제점과 연결
- 본 발명의 목적 명시
- 2~3개 과제로 구분 가능

**템플릿:**
```
본 발명은 상기와 같은 문제점을 해결하기 위하여 안출된 것으로서,
본 발명의 목적은 [과제 1]을/를 제공하는 것이다.
본 발명의 다른 목적은 [과제 2]을/를 제공하는 것이다.
```

---

### 5. 과제 해결 수단

**작성 원칙:**
- 청구항의 내용을 풀어서 설명
- 기술적 구성요소 나열
- "~하는 단계", "~하는 수단" 형식

**구조:**
```
상기 목적을 달성하기 위한 본 발명에 따른 [발명명칭]은/는,
[구성요소 1]과;
[구성요소 2]와;
[구성요소 3]을 포함하는 것을 특징으로 한다.
```

**예시:**
```
상기 목적을 달성하기 위한 본 발명에 따른 실행 계획 생성 시스템은,
사용자의 행동 패턴 데이터를 수집하는 데이터 수집부와;
상기 행동 패턴 데이터를 분석하여 최적 실행 시점을 예측하는 분석부와;
행동과학 이론에 기반하여 실행 계획을 생성하는 계획 생성부와;
상기 최적 실행 시점에 상기 실행 계획을 사용자에게 제공하는 출력부를
포함하는 것을 특징으로 한다.
```

---

### 6. 발명의 효과

**작성 원칙:**
- 종래 기술 대비 개선점
- 정량적 효과 포함 권장
- 각 과제에 대응하는 효과

**템플릿:**
```
본 발명에 따르면, 다음과 같은 효과가 있다.

첫째, [효과 1]
둘째, [효과 2]
셋째, [효과 3]
```

---

### 7. 도면의 간단한 설명

**작성 원칙:**
- 각 도면 번호별 설명
- 도면이 무엇을 나타내는지 명시

**템플릿:**
```
도 1은 본 발명의 일 실시예에 따른 시스템의 전체 구성도이다.
도 2는 본 발명의 일 실시예에 따른 처리 흐름도이다.
도 3은 본 발명의 일 실시예에 따른 사용자 인터페이스 예시도이다.
```

---

### 8. 발명을 실시하기 위한 구체적인 내용

**작성 원칙:**
- 가장 상세하게 작성하는 부분
- 도면 참조하며 설명
- 다양한 실시예 포함 가능
- 청구항 범위를 뒷받침

**구조:**
```
이하, 첨부된 도면을 참조하여 본 발명의 바람직한 실시예를
상세히 설명하기로 한다.

[시스템 구성 설명]
도 1을 참조하면, 본 발명에 따른 시스템(100)은...

[각 구성요소 상세 설명]
데이터 수집부(110)는...
분석부(120)는...
계획 생성부(130)는...

[동작 설명]
도 2를 참조하면, 먼저 단계 S210에서...

[변형 실시예]
다른 실시예에서, 상기 분석부는...
```

## 작성 팁

1. **청구항 먼저**: 청구항을 먼저 작성하고, 명세서에서 뒷받침
2. **도면 활용**: 복잡한 기술은 도면으로 설명
3. **일관된 용어**: 동일한 구성요소는 동일한 명칭 사용
4. **넓은 범위**: 실시예를 다양하게 하여 권리범위 확보
5. **선행기술 인용**: 배경기술에 선행기술 검색 결과 반영
