# 변리사 실무 기준 출원 전 최종 체크리스트

> 이 체크리스트는 실제 변리사 첨삭 피드백을 기반으로 작성되었습니다.
> 출원 전 모든 항목을 확인하세요.

## 1. 명세서 표현 검증

### 1-1. 권리범위 확장 표현

| 검증 항목 | 확인 |
|---------|-----|
| "~한다" → "~할 수 있다"로 변환했는가? | [ ] |
| "바람직하게는", "특히" 표현을 제거했는가? | [ ] |
| "바람직한 실시예" → "일 측면에 따른 일 실시예"로 변환했는가? | [ ] |

| 수치 효과는 【발명을 실시하기 위한 구체적인 내용】의 실시예에 있는가? | [ ] |
| 효과란은 "향상된다", "개선된다" 등 정성적 표현으로 작성했는가? | [ ] |

## 3. 청구항 검증 (기재불비 방지)

### 3-1. "상기" 참조 검증

| 검증 항목 | 확인 |
|---------|-----|
| 모든 "상기 ~~"가 해당 청구항 또는 인용 청구항에 기재되어 있는가? | [ ] |
| "상기 분석 결과"를 쓰려면 앞에 "분석 결과"가 정의되어 있는가? | [ ] |

**검증 방법:** 각 "상기" 앞에 Ctrl+F로 해당 용어 존재 확인

### 3-2. 주어 검증

| 검증 항목 | 확인 |
|---------|-----|
| 모든 종속항에 주어(상기 시스템은, 상기 방법은)가 있는가? | [ ] |
| "제N항에 있어서" 다음에 바로 동사가 오지 않는가? | [ ] |

```
❌ "제1항에 있어서, 복수 학습자의..."
✅ "제1항에 있어서, 상기 시스템은 복수 학습자의..."
```

### 3-3. 모호한 표현 검증

| 금지 표현 | 대체 표현 | 확인 |
|---------|---------|-----|
| 개인화된 | 사용자별 맞춤형, 각 사용자에게 맞춤화된 | [ ] |
| 적절한 | 구체적 기준 명시 또는 삭제 | [ ] |
| 효과적인 | 구체적 효과 명시 또는 삭제 | [ ] |

### 3-4. 용어 일관성 검증

| 검증 항목 | 확인 |
|---------|-----|
| 명세서에 정의한 용어를 청구항에서 동일하게 사용하는가? | [ ] |
| "스켈레톤 포인트" ↔ "관절점" 같은 혼용이 없는가? | [ ] |

## 4. 청구항 구조 검증

### 4-1. 1발명 1출원 검증 (특허법 제45조)

| 검증 항목 | 확인 |
|---------|-----|
| 시스템(장치) 청구항과 방법 청구항이 같은 출원에 있는가? | [ ] |
| 분리 출원 가능성을 인지하고 있는가? | [ ] |
| 거절시 분할출원 준비가 되어 있는가? | [ ] |

**권장:** 시스템 청구항(제1~N항)과 방법 청구항(제M~P항)을 별도 출원으로 분리

### 4-2. 프로세서+메모리 청구항 검증

| 검증 항목 | 확인 |
|---------|-----|
| "프로세서와; 메모리를 포함하는 장치" 형식 청구항에서 | [ ] |
| 프로세서 동작이 "단계"가 아닌 "구성"으로 기재되었는가? | [ ] |

```
❌ "프로세서는 ~를 추출하는 단계와; ~를 분석하는 단계를 수행"
✅ "프로세서는 ~를 추출하고; ~를 분석하도록 구성된"
```

## 체크리스트 사용 방법

1. 출원서 작성 완료 후 위 체크리스트를 **순서대로** 확인
2. 하나라도 [ ]가 남아있으면 **수정 후 재확인**
3. 모든 항목이 [x]가 되면 출원 진행
4. **불확실한 항목은 변리사 상담 권장**
