# 청구항 자동 생성 프롬프트 가이드

발명 설명을 입력하면 특허 청구항을 자동 생성하는 프롬프트 모음입니다.

---

## 기본 청구항 생성 프롬프트

### 장치 청구항 (시스템/장치 발명용)

```
당신은 특허 전문가입니다. 아래 발명 설명을 바탕으로 특허 청구항을 작성하세요.

## 발명 설명
{{발명_설명}}

## 작성 규칙
1. 독립항 1개 + 종속항 3-5개 구조
2. 독립항: "~를 포함하는 [발명의 명칭]" 형식
3. 종속항: "제N항에 있어서, ~를 특징으로 하는 [발명의 명칭]" 형식
4. 각 구성요소에 도면부호 (110, 120, 130...) 부여
5. 기술적 특징을 단계적으로 한정
6. 선행기술과 차별화되는 핵심 요소 강조

## 출력 형식
【청구항 1】
(독립항 내용)

【청구항 2】
제1항에 있어서, (종속항 내용)
```

### 방법 청구항 (소프트웨어/알고리즘용)

```
당신은 특허 전문가입니다. 아래 발명 설명을 바탕으로 방법 청구항을 작성하세요.

## 발명 설명
{{발명_설명}}

## 작성 규칙
1. "~하는 단계"로 각 단계 기술
2. 단계 순서를 논리적으로 구성
3. 각 단계에 부호 (S110, S120, S130...) 부여
4. 입력-처리-출력 흐름 명확히
5. 핵심 알고리즘/로직을 구체적으로 기술
6. "컴퓨터로 구현되는" 또는 "프로세서에 의해 수행되는" 명시

## 출력 형식
【청구항 1】
(a) ~하는 단계(S110);
(b) ~하는 단계(S120);
(c) ~하는 단계(S130);를 포함하는 [방법명].
```

## 고급 청구항 생성 프롬프트

### 다중 카테고리 청구항 세트

```
당신은 10년 경력의 변리사입니다. 다음 발명에 대해 완전한 청구항 세트를 작성하세요.

## 발명 정보
- 명칭: {{발명_명칭}}
- 기술분야: {{기술분야}}
- 핵심 기술: {{핵심_기술}}
- 선행기술 대비 차별점: {{차별점}}

## 작성 요구사항

### 1. 장치 청구항 (청구항 1-5)
- 독립항 1개: 전체 시스템/장치 청구
- 종속항 4개: 핵심 모듈별 상세 한정

### 2. 방법 청구항 (청구항 6-10)
- 독립항 1개: 전체 방법/프로세스 청구
- 종속항 4개: 핵심 단계별 상세 한정

### 3. 기록매체 청구항 (청구항 11)
- 컴퓨터 프로그램이 기록된 기록매체

## 청구항 작성 원칙
1. 독립항은 최대한 넓게 (광범위한 권리)
2. 종속항은 단계적으로 좁게 (구체적 실시예)
3. 선행기술 회피를 위한 신규 요소 강조
4. 심사관 거절에 대비한 대안적 표현 포함
5. 청구항 간 일관된 용어 사용
```

### AI/머신러닝 발명 전용

```
당신은 AI 분야 특허 전문가입니다. 다음 AI 발명의 청구항을 작성하세요.

## 발명 정보
- AI 모델: {{모델_종류}} (예: CNN, LSTM, Transformer)
- 입력 데이터: {{입력_데이터}}
- 출력 결과: {{출력_결과}}
- 학습 방법: {{학습_방법}}
- 응용 분야: {{응용_분야}}

## AI 특허 청구항 특수 규칙
1. "학습된 모델"이 아닌 "학습 방법" 또는 "추론 시스템" 청구
2. 모델 구조(레이어, 파라미터)를 구체적으로 기술
3. 기술적 효과(정확도 향상, 속도 개선 등) 명시
4. 데이터 전처리 단계 포함
5. 특징 추출 방법 상세히 기술

## 출력 구조
【청구항 1】 시스템 독립항 (학습 및 추론 시스템)
【청구항 2-4】 시스템 종속항 (모델 구조, 전처리, 후처리)
【청구항 5】 방법 독립항 (학습 방법)
【청구항 6-8】 방법 종속항 (학습 단계 상세)
【청구항 9】 방법 독립항 (추론 방법)
【청구항 10-11】 추론 종속항
```

## 특수 상황별 프롬프트

### 선행기술 회피 청구항

```
다음 선행기술을 회피하면서 발명을 보호하는 청구항을 작성하세요.

## 내 발명
{{내_발명_설명}}

## 회피해야 할 선행기술
1. {{선행기술_1}}: {{핵심_내용_1}}
2. {{선행기술_2}}: {{핵심_내용_2}}

## 차별화 전략
1. 선행기술에 없는 구성요소 강조
2. 구성요소의 결합 방식 차별화
3. 기술적 효과의 차이점 명시
4. 용어를 다르게 정의하여 권리범위 차별화

## 출력
- 선행기술 분석 (각 선행기술의 핵심 청구항)
- 차별화 포인트 (내 발명만의 신규 요소)
- 회피 청구항 (선행기술과 겹치지 않는 표현)
```

### 거절 대응 보정 청구항

```
다음 거절이유에 대응하여 청구항을 보정하세요.

## 원청구항
{{원청구항_내용}}

## 거절이유
{{거절이유_내용}}
- 유형: [신규성/진보성/기재불비/청구범위 불명확]
- 인용 선행기술: {{인용_선행기술}}

## 보정 전략
1. 거절이유 유형에 따른 대응:
   - 신규성: 차별화 구성요소 추가
   - 진보성: 효과/결합의 용이성 주장
   - 기재불비: 명세서 내 근거 확보
   - 불명확: 용어 정의 추가

2. 보정 원칙:
   - 최초 명세서 범위 내에서만 보정
   - 청구범위를 과도하게 좁히지 않음
   - 종속항 병합으로 독립항 강화

## 출력
- 보정 청구항
- 보정 이유 (의견서용 설명)
```

## 실전 예시

### 예시 1: 동작 분석 AI 시스템

**입력:**
```
발명: MediaPipe로 관절 좌표 추출 후 Bi-LSTM으로 동작 분석하는 시스템
특징: 시간적 패턴 학습, 양방향 분석, 실시간 피드백
```

**출력 청구항:**
```
【청구항 1】
사용자의 동작 영상을 수신하는 영상 수신부(110);
상기 영상 수신부로부터 수신된 영상에서 복수의 관절 좌표를 추출하는
포즈 추정부(120);
상기 추출된 관절 좌표의 시간적 시퀀스를 양방향 장단기 메모리
네트워크(Bi-LSTM)로 분석하여 동작 패턴을 인식하는 동작 분석부(130); 및
상기 인식된 동작 패턴에 기반하여 실시간 피드백을 생성하는
피드백 생성부(140);를 포함하는 인공지능 기반 동작 분석 시스템.

【청구항 2】
제1항에 있어서,
상기 포즈 추정부(120)는 MediaPipe 프레임워크를 이용하여
33개 이상의 신체 관절 좌표를 추출하는 것을 특징으로 하는
인공지능 기반 동작 분석 시스템.

【청구항 3】
제1항에 있어서,
상기 동작 분석부(130)는:
순방향 LSTM 레이어;
역방향 LSTM 레이어; 및
상기 순방향 및 역방향 출력을 결합하는 결합 레이어;를 포함하는 것을
특징으로 하는 인공지능 기반 동작 분석 시스템.
```

## 청구항 품질 체크리스트

생성된 청구항을 아래 기준으로 검토하세요:

### 형식 검토
- [ ] 독립항이 단일 문장인가?
- [ ] 종속항이 올바른 형식(제N항에 있어서)인가?
- [ ] 도면부호가 일관되게 사용되었는가?
- [ ] 용어가 명세서와 일치하는가?

### 내용 검토
- [ ] 독립항이 발명의 핵심을 포함하는가?
- [ ] 종속항이 단계적으로 구체화되는가?
- [ ] 선행기술과 차별화되는 요소가 명확한가?
- [ ] 기술적 효과가 암시되어 있는가?

### 전략 검토
- [ ] 권리범위가 적절한가? (너무 좁거나 넓지 않은가)
- [ ] 회피 설계가 어려운 표현인가?
- [ ] 거절 시 보정 여지가 있는가?
- [ ] 실시예가 청구항 범위에 포함되는가?

## ⚠️ 변리사 실무 기준 최종 검토 체크리스트

생성된 청구항을 **출원 전** 반드시 아래 기준으로 최종 검토하세요:

### 1. "상기" 참조 검증 (기재불비 방지)
- [ ] 모든 "상기 ~~"가 **해당 청구항 또는 인용 청구항에 기재**되어 있는가?
- [ ] "상기 분석 결과"를 쓰려면 앞에 "분석 결과"가 정의되어 있는가?

```
✅ 검증 방법: 각 "상기"마다 Ctrl+F로 해당 용어가 위에 있는지 확인
```

### 2. 주어 검증 (기재불비 방지)
- [ ] 모든 종속항에 **주어(상기 시스템은, 상기 방법은)**가 있는가?
- [ ] "제N항에 있어서" 다음에 바로 동사가 오면 **주어 누락**

```
❌ "제1항에 있어서, 복수 학습자의..." (주어 없음)
✅ "제1항에 있어서, 상기 시스템은 복수 학습자의..."
```

### 3. 모호한 표현 검증 (기재불비 방지)
- [ ] "개인화된", "적절한", "효과적인" 등 **정의 없는 형용사** 사용 안 했는가?
- [ ] 모호한 표현은 **구체적 정의**로 교체

```
❌ "개인화된 피드백"
✅ "사용자별 맞춤형 피드백" 또는 "각 사용자의 수행 기록에 기반한 피드백"
```

### 4. 1발명 1출원 검증 (분리 출원 대비)
- [ ] 시스템(장치) 청구항과 방법 청구항이 **같은 출원**에 있는가?
- [ ] 심사관 재량에 따라 **분리 출원 요구** 가능성 인지

```
📌 권장: 시스템 청구항(제1~N항)과 방법 청구항(제M~P항)을
   별도 출원으로 분리하거나, 거절시 분할출원 준비
```

### 5. 프로세서+메모리 청구항 검증
- [ ] "프로세서와; 메모리를 포함하는 장치" 형식의 청구항에서
- [ ] **프로세서 동작이 방법(단계)으로 기재**되어 있지 않은가?

```
❌ "프로세서는 ~를 추출하는 단계와; ~를 분석하는 단계를 수행" (방법+물건 혼합)
✅ "프로세서는 ~를 추출하고; ~를 분석하도록 구성된" (물건으로 통일)
```

### 6. 명세서 뒷받침 검증
- [ ] 모든 청구항 구성요소가 **명세서에 설명**되어 있는가?
- [ ] 청구항에서 새로 정의한 용어가 명세서에 없으면 **거절 사유**

### 7. 용어 일관성 검증
- [ ] "스켈레톤 포인트", "관절 좌표" 등 **명세서에서 정의한 용어**를 청구항에서 동일하게 사용하는가?
- [ ] 명세서: "스켈레톤 포인트" → 청구항: "스켈레톤 포인트" (동일해야 함)

## 변리사 수준 청구항 생성 프롬프트

### 최종 검토 포함 버전

```
당신은 15년 경력의 한국 변리사입니다. 다음 발명의 청구항을 작성하되,
**출원 전 변리사 검토 기준**을 모두 충족하세요.

## 발명 정보
{{발명_설명}}

## 필수 검증 사항 (하나라도 위반시 수정)

1. **"상기" 검증**: 모든 "상기 ~~"는 해당 청구항 또는 인용 청구항에 기재된 것만 참조
2. **주어 검증**: 모든 종속항에 주어(상기 시스템은, 상기 방법은) 포함
3. **모호한 표현 금지**: "개인화된", "적절한", "효과적인" 등 사용 금지
4. **명세서 용어 일치**: 명세서에 정의된 용어와 동일하게 사용
5. **프로세서 청구항**: 방법(단계)이 아닌 물건(구성)으로 기재

## 출력 형식
【청구항 1】 (독립항 - 시스템/장치)
【청구항 2~N】 (종속항)
【청구항 M】 (독립항 - 방법) ← 별도 출원 권장 표시
【청구항 P】 (기록매체/장치) ← 별도 출원 권장 표시

## 검증 결과
각 청구항에 대해 위 5가지 검증 결과를 ✅/❌로 표시
```

## 자주 사용하는 표현

### 시스템/장치 표현
- "~를 포함하는 시스템/장치"
- "~와 연결된 ~부"
- "~를 수행하도록 구성된 프로세서"
- "~가 저장된 메모리"

### 방법 표현
- "~를 수신/획득/입력받는 단계"
- "~를 분석/처리/연산하는 단계"
- "~를 기반으로 ~를 생성/출력하는 단계"
- "~를 학습/훈련하는 단계"

### AI/ML 표현
- "학습된 신경망 모델을 이용하여"
- "딥러닝 기반 ~"
- "~를 입력으로 하고 ~를 출력으로 하는"
- "사전 학습된 모델을 파인튜닝하여"
